    Section 10.1 Context Free Grammars (CFGs)

    Subsection Regular Languages Review

    To best understand CFGs, we must understand how they differ from Regular Languages. Regular Languages:
    • have no memory - they use states
    • have modular looping - they use state transitions
    • can be expressed in Regular Expressions, Deterministic Finite Automata, and Non-Deterministic Finite Automata
    In order to build upon Regular Languages, we move to constructions that allow some limited forms of matching. Context Free Languages come about when Regular Languages have access to stack memory, and they are expressed through Context Free Grammars.

    Subsection Basics of Context Free Grammars

    Context Free Grammars (CFGs) consist of a series of rules or productions.
    \(CFG_1:\)
    \(S \rightarrow aSb|T\)
    \(T \rightarrow cTd|cd\)
    Each line of a CFG is called a rule. The left hand side of a rule is always a variable. Variables are denoted by uppercase letters. The right hand side of a rule is some combination terminals and variables. Terminals can be elements of the alphabet of the Context Free Language or the empty string (\(\epsilon\)). They are denoted by lower case letters.
    CFGs have a start variable, it is usually on the left hand side of the first rule and is denoted with \(S\text{.}\) To produce strings, variables are replaced by one of the options avaliable on the production’s right hand side. In a production rule, \(|\) is used to mean OR. The left hand side can be replaced by one of the right hand side.
    Here is a sample derivation from \(CFG_1\text{:}\)
    \(S \rightarrow aSb\) \(S\) is replaced by \(aSb\)
    \(aSb \rightarrow aTb\) The \(S\) in \(aSb\) is relapced with \(T\)
    \(aTb \rightarrow acTdb\) The \(T\) in \(aTb\) is replaced with \(aTd\)
    \(acTdb \rightarrow accddb\) \(T\) in \(acTdb\) is replaced by \(cd\)
    This makes the final string: \(accddb\text{.}\)

    Exploration 1.

    Consider the following grammar:
    \(CFG_2\text{:}\)
    \(S \rightarrow aS | \epsilon\)
    What strings can we make from it?
    \(\epsilon\text{:}\)
    \(S \rightarrow \epsilon\) \(S\) is replaced by \(\epsilon\)
    \(a\text{:}\)
    \(S \rightarrow aS\) \(S\) is replaced by \(aS\)
    \(aS \rightarrow a\epsilon\) \(S\) is replaced by \(\epsilon\)
    \(aa\text{:}\)
    \(S \rightarrow aS\) \(S\) is replaced by \(aS\)
    \(aS \rightarrow aaS\) \(S\) is replaced by \(aS\)
    \(aaS \rightarrow aa\) \(S\) is replaced by \(\epsilon\)
    And many more strings can be made! But, what languge is this?
    The language of this grammar is any amount of \(a\)s concatenated.

    Exploration 2.

    Consider the following grammar:
    \(CFG_3\text{:}\)
    \(S \rightarrow 0S1 | \epsilon\)
    What strings can we make from it?
    Answer.
    \(\epsilon\text{:}\)
    \(S \rightarrow \epsilon\) \(S\) is replaced by \(\epsilon\)
    \(01\text{:}\)
    \(S \rightarrow 0S1\) \(S\) is replaced by \(0S1\)
    \(0S1 \rightarrow 0\epsilon 1\) \(S\) is replaced by \(\epsilon\)
    \(0011\text{:}\)
    \(S \rightarrow 0S1\) \(S\) is replaced by \(0S1\)
    \(0S1 \rightarrow 00S11\) \(S\) is replaced by \(0S1\)
    \(00S11 \rightarrow 00\epsilon 11\) \(S\) is replaced by \(\epsilon\)
    And many more strings can be made! But, what languge is this?
    The language of this grammar is any amount of 0s followed by the same amount of 1s.

    Exploration 3.

    Let us consider a grammar with multiple variables.
    \(CFG_4\text{:}\)
    \(S \rightarrow aS | T | \epsilon\)
    \(T \rightarrow bT | \epsilon\)
    Can we generate only \(a\)s?
    \(S \rightarrow aS\) \(S\) is replaced by \(aS\)
    \(aS \rightarrow aaS\) \(S\) is replaced by \(aS\)
    \(aaS \rightarrow aa\epsilon\) \(S\) is replaced by \(\epsilon\)
    The resulting string is \(aa\text{.}\) Therefore, \(CFG_4\) can generate only \(a\)s.
    Can we generate only \(b\)s? How?
    Answer.
    \(S \rightarrow T\) \(S\) is replaced by \(T\)
    \(T \rightarrow bT\) \(T\) is replaced by \(T\)
    \(bT \rightarrow bbT\) \(T\) is replaced by \(bT\)
    \(bbT \rightarrow bb\epsilon\) \(T\) is replaced by \(\epsilon\)
    The resulting string is \(bb\text{.}\) Therefore, \(CFG_4\) can generate only \(b\)s.
    What are some other strings that we can generate using \(CFG_4\text{?}\)
    • \(\displaystyle ab\)
    • \(\displaystyle aaabb\)
    • \(\displaystyle abbbbbb\)
    The language of this grammar is the empty string or any amount of \(a\)s followed by any amount of \(b\)s.

    Subsection Creating CFGs from Regular Expressions

    Any Regular Expression is a Context Free Language and can easily be implemented into a CFG using the bellow framework:
    • union is an \(|\text{,}\) another rule
    • concatenation is a \(V \rightarrow V_1v_2\) rule
    • * is a \(S \rightarrow stringS | \epsilon\) rule
    Let us look a little ddeper into matching with another exploration.
    Exploration 4.
    Construct a CFG for \(L_5\) where
    \begin{equation*} L_5 = \{a^i(bc)^i | i \geq 0 \} \end{equation*}
    Before looking at the correct grammar, let’s first analyze some common incorrect answers. \(CFG_5'\) below demonstrates one common mistake:
    \(CFG_5'\text{:}\)
    \(S \rightarrow ASB | \epsilon\)
    \(A \rightarrow aA | \epsilon\)
    \(B \rightarrow bcB | \epsilon\)
    But, why is \(CFG_5'\) incorrect? Construct some strings using this grammar.
    • abc
    • aabcbc
    • aabcbc
    • aaaaaaa
    But, how did the string \(abcbc\) come about from \(CFG_5'\text{?}\)
    \(S \rightarrow ASB\) \(S\) is replaced by \(ASB\)
    \(ASB \rightarrow aSB\) The \(A\) is replaced by \(a\)
    \(aSB \rightarrow aASBB\) The \(S\) is replaced by \(ASB\)
    \(aASBB \rightarrow aaSBB\) The \(A\) is replaced by \(a\)
    \(aaSBB \rightarrow aaBB\) The \(S\) is replaced by \(\epsilon\)
    \(aaBB \rightarrow aabcB\) The \(B\) is replaced by \(bc\)
    \(aabcB \rightarrow aabc\) The \(B\) is replaced by \(\epsilon\)
    By not forcing the \(a\) and \(bc\) to be created at the same time and inculding an epsilon, then it is possible to a string to be created that is not in the language.
    Below is the correct grammar for \(L_5\text{.}\)
    Answer.
    \(CFG_5\text{:}\)
    \(S \rightarrow aSbc | \epsilon\)
    We can see that matched pairs must be contained in the same rule.
    Following our thinking from the exploration, we can see that interleaved matching does not work within Context Free Grammars. An example is this would be a a language where
    \begin{equation*} L = \{ 0^n1^m0^n1^m\} \end{equation*}
    Matching three groupings also does not work. An example of this would be:
    \begin{equation*} L = \{0^n1^n0^n\} \end{equation*}
    Example 10.1.
    Here is an example that is applicable to more general computer science. Imagine parsing a computer program. You have run a REX which returns these terminals:
    b boolean expressions
    i an \(if\) keyword
    c a code statement
    e an \(else\) keyword
    f an \(elif\) keyword
    : a colon
    Write the CFG using \(b\text{,}\)\(i\text{,}\)\(c\text{,}\)\(e\text{,}\)\(f\text{,}\) and \(:\) for Python \(if\) statements. You may ignore returns and tabbing.
    Answer 1.
    \(CFG\text{:}\)
    \(S \rightarrow iB:CFE\)
    \(B \rightarrow bB | b\)
    \(C \rightarrow cC | c\)
    \(F \rightarrow fB:C | FF | \epsilon\)
    \(E \rightarrow e:C | \epsilon\)
    The CDFG can also be written as seen below.
    Answer 2.
    \(CFG\text{:}\)
    \(S \rightarrow ib:U_1U_2U_3 | \epsilon\)
    \(U_1 \rightarrow cU_1 | c\)
    \(U_2 \rightarrow fB:U_1U_2 | \epsilon\)
    \(U_3 \rightarrow e:U_1 | \epsilon\)

    Section 10.2 Parse Trees

    Derivations are often drawn as a tree so that precedence and matching is clean. We will consider \(CFG_1\) for this example.
    \(CFG_1:\)
    \(S \rightarrow aSb|T\)
    \(T \rightarrow cTd|cd\)
    We list the string at the bottom of the tree with the starting variable at the top. For the first example we will follow the derrivation that creates \(accddb\text{.}\) It is repeated below and has been numbers for convinence:
    1) \(S \rightarrow aSb\) 2) \(aSb \rightarrow aTb\)
    \(S\) is replaced by \(aSb\) The \(S\) in \(aSb\) is relapced with \(T\)
    3) \(aTb \rightarrow acTdb\) 4) \(acTdb \rightarrow accddb\)
    The \(T\) in \(aTb\) is replaced with \(aTd\) \(T\) in \(acTdb\) is replaced by \(cd\)
    Here, you can see the \(c\) and \(d\) matching and the \(a\) and \(b\) matching a little more clearly.

    Finally, we will need \(E\) to give us \(0\)

    Section 10.3 Ambiguity

    Abmibuous grammars can generate some string in two or more different ways. The grammar as shown in the exploration above is ambiguous as there is more than one way to generate the string, but the language this grammar represents does have a grammar that is not ambiguous.
    If we want to force some matching to have precedence, you simply push the items with higher precedence to be matched by rules further down the grammar. Consider \(CFG_5\) once again.
    \(CFG_5:\)
    \(E \rightarrow E+E | E*E | 0 | 1\)
    Here, \(E*E\) is of lower precedence than \(E+E\text{,}\) so it can be moved down.
    \(CFG_{5a}:\)
    \(E \rightarrow E+E | T | 0 | 1\)
    \(T \rightarrow E*E\)
    The \(0 | 1\) also has lower precedence than \(E+E\) and \(E*E\text{,}\) but there needs to be a way to generate those. They can also be given their own variable.
    \(CFG_{5b}:\)
    \(E \rightarrow E+E | T\)
    \(T \rightarrow E*E | N\)
    \(N \rightarrow 0 | 1\)
    Using the above grammar, there is now only one way to create the string \(1+1*0\text{.}\) This is the parse tree:
    This is the only possible way to create the parse tree that generates \(1+1*0\) from \(CFG_{5b}\) because the "+" must be matched first.
    Some languages have only ambiguous grammars. These languages are called interherently ambiguous. The language below is an example of an inherently ambiguous language:
    \begin{equation*} L_6 = \{a^ib^jc^k | i=j \mbox{ or } j=k\} \end{equation*}
    \(CFG_6\) is one possible grammar for \(L_6\text{:}\)
    \(CFG_6:\)
    \(S \rightarrow AU_1 | U_2G\)
    \(A \rightarrow aAb | \epsilon\)
    \(U_1 \rightarrow cU_1 | \epsilon\)
    \(U_2 \rightarrow aU_2 | \epsilon\)
    \(G \rightarrow bGc | \epsilon\)
