    Section 18.1 Introduction to Reducibility

    Reducibility is a way to show that two problems or questions are the same with the goal of having one problem be easier than the other. In general, we will take a problem,\(B\text{,}\) and use it to show a solution to another problem,\(A\text{.}\) From here on out, we will use \(!Td\) to mean that something is not Turing decidable. We will also use \(!Tr\) to mean that something is not Turing recognizable.

    Exploration 3.

    Consider the following problem: Find the areas of a square.
    What is one way to solve this problem?
    Answer 1.
    Length times width! We could solve the problem by finding the width and squaring it.
    What is another way to solve this problem?
    Hint.
    Don’t put too much thought into this.
    Answer 2.
    Another option is to find an Oracle! They would be able to tell us the area of the square.
    There is obviously one of these solutions that is much easier to reasonably accomplish than the other one. That is our goal, to find an easier way to reduce a given problem to the easiest, quickest problem to solve.
    If a problem,\(A\text{,}\) reduces to problem,\(B\text{,}\) then \(A\) is not harder than \(B\text{,}\) and \(B\) is not easier than \(A\text{.}\) If \(A \leq m B\) and \(A\) is undecidable then \(B\) is undecidable. The symbol \(\leq m\) is being used to represent "reduces to".

    Exploration 4.

    Consider:
    \begin{equation*} A_{TM} \leq H_{TM} \end{equation*}
    In other words, does the Acceptance Turing Machine reduce to the Halting Turing Machine?
    I can "solve" (decide) \(A_{TM}\) in the presence of a decider for \(H_{TM}\text{.}\)
    \begin{equation*} H_{TM} = \{ \langle M,w \rangle | M \mbox{ halts on } w\} \end{equation*}
    \begin{equation*} A_{TM} = \{\langle M,w \rangle | M \mbox{ accepts } w \} \end{equation*}
    1. Assume we have a decider, \(R\text{,}\) for \(H_{TM}\text{.}\)
    2. Given \(R\text{,}\) I can decide \(A_{TM}\text{,}\) which is a contradiction because \(A_{TM}\) is undecidable.
    Solving \(H_{TM}\) allows us to solve \(A_{TM}\text{.}\) So, \(H_{TM}\) is at least as complex as \(A_{TM}\text{,}\) because if it were easier, \(A_{TM}\) would be too. Thus, \(H_{TM}\) is undecidable.
    The above exploration is not how we will do most reductions, but it is a great introduction to the idea.

    Section 18.2 Proving Reducibility

    We will first walk through an example of a reducibility proof.

    Exploration 5.

    Consider:
    \begin{equation*} Regular_{TM} = \{ \langle m \rangle | L(m) \mbox{ is a RL}\} \end{equation*}
    We then come up with the diagram below:
    Where:
    Q
    a TM constructed from \(M\) and \(w\text{,}\) and accepts on input of \(0^n1^n\)
    else
    Run \(M\) on \(w\text{,}\) if \(M\) accepts, accept!
    This had been a proof sketch.
    We will now go through a proof outline and a full proof.
    Outline:
    • We want to show \(Regular_{TM}\) is undecidable.
    • We know that \(A_{TM}\) is undecidable.
    • If we can show that \(A_{TM}\) reduces to \(Regular_{TM}\text{,}\) then we will know that it is undecidable (because if it weren’t, the reduction would allow \(A_{TM}\) to be decidable).
    We will now look at a full example proof. Note that the itallicized language is largely epositiory and set up language.
    1. Assume \(Regular_{TM}\) is decidable and there is some machine \(R\) that decides it. \(R\text{,}\) when handed a TM, will tell us if the TM’s langbuage is regular decidably.
    2. We show \(A_{TM}\) reduces to \(Regular_{TM}\text{.}\)
    3. We create a machine, \(Q\text{,}\) using the \(M\) and \(w\) imputs from \(A_{TM}\text{.}\) \(Q\) takes input \(x\text{.}\)
      • If \(x\) is in \(0^n1^n\text{,}\) \(Q\) accepts.
      • Otherwise, \(Q\) runs \(M\) on \(w\text{.}\) If \(M\) accepts, \(Q\) accepts.
        This machine, Q, is special.
        • If M accepts w, Q accepts all inputs x. This is a regular language.
        • If M does not accept w, Q accepts only inputs that match \(0^n1^n\text{.}\) This is NOT a regular language.
    4. We now take our machine,\(Q\text{,}\) and pass it over to our decider, \(R\text{.}\) \(R\) will tell us if the language of \(Q\) is regular or not. If it is regular, \(M\) accepts \(w\text{.}\) If it is not, then \(M\) does not accept. We’ve built an \(A_{TM}\) decider.
    5. So, we’ve taken inputs to \(A_{TM}\text{,}\) crafted a machine, \(Q\text{,}\) passed it to \(R\text{,}\) and found that \(R\) somehow helps us decide \(A_{TM}\text{.}\)
    6. We know \(A_{TM}\) is not decidable, so this is a contradition.
    7. Therefore, our assumption that \(Regular_{TM}\) was decidable must have been incorrect.

    Exercises 18.3 Exercises

    1.

    Prove that \(E_{TM}\) is decidable.
    \begin{equation*} E_{TM} = \{\langle M \rangle | L(M) \mbox{ is } \emptyset \} \end{equation*}
    Hint.
    \(Q(x): \)
    • If \(x != w\) then \(Q\) rejects
    • \(M\) accepts \(w : Q\) rejects
    • else \(Q\) accepts \(x\text{.}\)
    \begin{equation*} L(Q) = \emptyset , Q \in E_{TM} \mbox{ when }M \mbox{ accepts } w \end{equation*}
    \begin{equation*} L(Q) = \{x\}, Q \notin E_{TM} \mbox{ when }M \mbox{ does not accept }w \end{equation*}

    2.

    Prove that \(CFL_{TM}\) is decidable.
    \begin{equation*} CFL_{TM} = \{\langle M \rangle | L(M) \mbox{ is a CFL} \} \end{equation*}
    Hint.
    General idea: Build a machine, \(Q\text{,}\) such that when we pass it to \(R\) (the decider for the language in question), \(R\) accepts \(Q\) when \(M\) accepts \(w\) (and doesn’t when it doesn’t).

    3.

    Prove the following language is undecidable by using reducibility and \(A_{TM}\text{.}\)
    \begin{equation*} L = \{\langle M, w \rangle | M \mbox{ is a Tm that moves the head on input } w\} \end{equation*}

    4.

    Consider the TM below:
    \begin{equation*} CONFIG_{TM} = \mbox{\{This machine, M, is a TM that enters a state with its head over the} \end{equation*}
    \begin{equation*} \mbox{same cell AND the whole tape is the same as the last time the TM was in this state.\}} \end{equation*}
    \(CONFIG_{TM}\) is undecidable and we can prove this with a reduction. Show that \(A_{TM} \leq _m CONFIG_{TM}\text{.}\)
