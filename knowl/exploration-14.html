So, if our DFA has \(n\) states, then any string with length at least \(n\) will revisit some state that the DFA has been in before. This is an application of the Pigeon Hole Principle. A string with \(n\) symbols needs \(n+1\) states if it does not want to revisit a state.
Writing out these tables can make looping patterns more apparent. In \(M_1\text{,}\) the transition among \(B\) to \(C\) and \(C\) to \(B\) could be repeated by a string. Another loop that can be repeated is \(A\) to \(A\text{.}\) Loops can be removed and still have a string in the language.
We can see this using the first string,
\begin{equation*} s^{(1)} = 110101111011\text{.} \end{equation*}
The repeating \(1\)s at the beginning can be removed and the string is still in the languge, so the condensed string is
\begin{equation*} s^{(1)'} = 0101111011 \end{equation*}
The repeating \(10\)s resulting from \(B\) to \(C\) transitions can also be taken out.
\begin{equation*} s^{(1)''} = 011111 \end{equation*}
This string is also in the language.
