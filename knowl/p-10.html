For example, characters \(a\text{,}\) \(b\text{,}\) and \(c\) are all included in alphabet \(A\text{.}\)
\begin{equation*} A = \{a, b, c\} \end{equation*}
A language will consist of strings created from this alphabet. Order matters. For example \(S\) and \(Q\) are strings from alphabet \(A\text{.}\)
\begin{equation*} S = baba \end{equation*}
\begin{equation*} T = aaaaa \end{equation*}
these example strings may be included in language \(L_1\text{.}\) You might say:
\begin{equation*} S \in L_1 \end{equation*}
which means \(S\) is in the language \(L_1\text{.}\) \(L_1\) might have other strings in it as well. We would normally list those in one form of set notation if required. If \(L_1\) consisted of just the two strings above, we could fully list them like this:
\begin{equation*} L_1 = \{baba, aaaaa\} \end{equation*}
A binary string (like \(1010\)) would require a binary alphabet (\(\{0,1\}\)). A sample binary language might be:
\begin{equation*} \{w | w\ \text{begins with a}\ 1\} \end{equation*}
This language consists of any binary string that begins with a 1. So some sample strings in the language are: 1, 10, 11, 100, 101, 110, 111, 1000, 1001 (and infinitely more). The following strings would not be in the language: 0, 01, 001, 010, 011, 0100, 0101 (and also infinitely more).
