    So, if our DFA has \(n\) states, then any string with length at least \(n\) will revisit some state that the DFA has been in before. This is an application of the Pigeon Hole Principle. A string with \(n\) symbols needs \(n+1\) states if it does not want to revisit a state.
    Writing out these tables can make looping patterns more apparent. In \(M_1\text{,}\) the transition among \(B\) to \(C\) and \(C\) to \(B\) could be repeated by a string. Another loop that can be repeated is \(A\) to \(A\text{.}\) Loops can be removed and still have a string in the language.
    We can see this using the first string,
    \begin{equation*} s^{(1)} = 110101111011\text{.} \end{equation*}
    The repeating \(1\)s at the beginning can be removed and the string is still in the languge, so the condensed string is
    \begin{equation*} s^{(1)'} = 0101111011 \end{equation*}
    The repeating \(10\)s resulting from \(B\) to \(C\) transitions can also be taken out.
    \begin{equation*} s^{(1)''} = 011111 \end{equation*}
    This string is also in the language.
    Try a couple more strings to identify patterns. Are there any similarly deduced strings that you can find?
    Loops can be taken out, or they can be taken many times.
    The Pigeon Hole Principle can be seen in all DFAs. And since it can be, we can build it into a mechanism to help us understand when languages are regular. That brings us to the Pumping Lemma.

    Section 9.2 Spotting Loops and Repetition in FAs

    Consider \(M_2\) where
    \begin{equation*} L(M_2) = L_2 \end{equation*}
    \begin{equation*} \Sigma = {0,1} \end{equation*}
    Consider the following strings as given to \(M_2\text{.}\)
    • \(s^{(0)} = 01:\)
      States A B C
      String 0 1
    • \(s^{(1)} = 0111:\)
      States A B C C C
      String 0 1 1 1
    • \(s^{(2)} = 01001001:\)
      States A B C C C C C C C
      String 0 1 0 0 1 0 0 1
    After writing out these parce tables, it is clear to see that any combination of characters after an initial \(01\) can be repeated without affecting if the string is accepted. This not only allows us to determine that \(L_2 = \{\mbox{any string that begins with 01}\}\) but also helps us understand that any concatenation of characters \(0\text{,}\) \(1\text{,}\) \(01\text{,}\) etc. can be reapeated any amount of times. Even \(P\) number of times.

    Section 9.3 The Pumping Lemma for Regular Languages

    The Pumping Lemma for Regular Languages can be used to prove that languages are regular. It follows the general form outlined below.
    If \(L\) is regular, then it has a pumping length \(p\text{.}\) Every string in \(L\) of length at least \(p\) will be pumpable.
    Given this string, call it \(s\) (\(S^{(1)}\)). It has three parts:
    • A beginning that can be \(\epsilon\) - call it \(x\)
    • A pumpable part that cannot be \(\epsilon\) - call it \(y\)
    • An end that can be \(\epsilon\) - call it \(z\)
    The key here is that we can make new strings in the language by removing \(y(s^{(1)})\) or adding \(y\) as many times as we’d like. Such as: \(s^{(2)}\text{,}\) \(s^{(3)}\text{,}\) \(s^{(4)}\text{,}\) etc.
    This works for any string that is long enough, and it happens within the first \(p\) symbols of the input string.
    In general, if the language of a machine is regular and the test string has length of at least p, then it is pumpable. This also means that there is some non-empty \(y\) loop, but this will typically be written more formally and closer to:
    1. If a language is a Regular Language then
    2. \(\exists p \geq 1\) such that \(\forall x \in L\) with \(|s|\geq p\)
    3. Then \(s^{(1)} = xyz\) where:
      \begin{equation*} |y| > 0 \end{equation*}
      \begin{equation*} |xy| \leq p \mbox{, and } \end{equation*}
      \begin{equation*} \forall i \geq 0, xy^iz\in L \end{equation*}
      \begin{equation*} (\mbox{also known as: }) s^{(i)}\in L \end{equation*}

    Theorem 9.4.

    Take a languge we want to prove is not regular.
    1. Assume it is regular.
    2. Pick a string that is at least of length \(p\) in the language
    3. Show that every possible choice for \(y\) cannot work
    4. Thus the assumption is wrong and by contradition, the language is not regular.

    Exploration 15. Pumping Lemma for Regular Languages without a Machine.

    Say \(L_1\text{:}\) has a number of 0s divisible by 3 and an even number of 1s.
    • \(L_1\) is a Regular Language.
    Let’s say we don’t know \(p\text{,}\) just that it exists. Let’s pick a string that’s \(p\) long.
    • This staisfies \(s\in L\) and \(|s|\geq p\)
    \begin{equation*} s^{(1)} = (00011)^p \end{equation*}
    Some piece of \(s^{(1)}\) in the first \(p\) symbols is pumpable.
    • There is some \(y\) that we can extract or insert as many times as we want

    Exploration 16. Pumping Lemma for Regular Languges with Defined Language.

    Consider \(L_{01}=\{0^n1^n|n\leq 0 \}\text{.}\)
    1. Assume \(L_{01}\) is regular thus obeying the Pumping Lemma for Regular Languages.
    2. Let us choose \(s^{(1)} = 0^p1^p\) since \(L_{01}\) is a Regular Language, any string that is long enough should be pumpable.
    3. A pumpable part (\(y\)) must occur in the first \(p\) symbols. So, \(y\) must contain one or more 0s and no 1s since the first \(p\) symbols are 0s.
      If we pump down to \(s^{(0)}\text{,}\) we will thus remove some 0s without removing any 1s. \(s^{(0)}\) cannot possibly be in \(L_{01}\text{,}\) no matter the choice for \(y\text{.}\)
    4. Thus the assumption is wrong and by contradiction, the language is not regular.

    Section 9.4 Additional Non-Regular Languages for Proving

    Here are some additional, fun, non-regular languages to try to prove (in increasing complexity):
    • \(\displaystyle \{0^i1^j|i>j\}\)
    • \(\displaystyle \{0^i1^j|i\geq j\}\)
    • \(\displaystyle \{0^i1^j|i \lt j\}\)
    • \(\displaystyle \{0^i1^j|i \leq j\}\)
    • \(\displaystyle \{0^n1^m0^n\}\)
    • \(\displaystyle \{a^ib^jc^k|i=j \mbox{ when } k=1\}\)
    • \(\displaystyle \{ww^R|w \mbox{ is a binary string}\}\)
    • \(\displaystyle \{w|w \mbox{ is a palindrome}\}\)
    • \(\displaystyle \{ab^nc^n\}\)
    • \(\displaystyle \{a^ib^jc^k| j = k \mbox{ when } i = 1 \}\)
    Unless stated otherwise, \(i,j,k,m,n \geq 0\text{.}\) You can try \(>0\) or \(>2\) for additional practice.
