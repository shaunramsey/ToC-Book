    Chapter 8 Closure and Regular Languages

    Objectives

    1. Discrete Math Review
    2. Closure

    Section 8.1 Closure

    A set is closed under an operation when any application of that operation on an element (or elements) in that set always results in an element in the set.

    Exploration 7.

    Each column is an element of the set.
    1. If I take an element and swap its top character with the top character of another element, do I stay in the set?
    2. How about swapping bottom characters?
    3. What if I change the bottom character to a \(\triangle\text{?}\)
    4. How about swapping \(\square\)s and \(\bigcirc\)s?
    5. How about swapping middle characters to \(\triangle\)s, \(\square\)s, or \(\bigcirc\)s?

    Exploration 8. Closure in Natural Numbers.

    Consider a few simple operations applied to the natural numbers.
    \begin{equation*} \N = \{1,2,3,4,5,6,7,8...\} \end{equation*}
    • Addition - take any two \(\N\) and add them. Is the result always in \(\N\text{?}\)
      • If yes, \(\N\) are closed under addition
      • If no, \(\N\) are not closed under addition and a counter-example needs to be provided
      Answer.
      Yes.
    • Subtraction - take any two \(\N\) and subtract one from the other. Is the result always in \(\N\text{?}\)
      • If yes, \(\N\) are closed under subtraction
      • If no, \(\N\) are not closed under subtraction and a counter-example needs to be provided
      Answer.
      No. Negative numbers are not included in the \(\N\text{.}\)
      \begin{equation*} 1,2 \in \N \end{equation*}
      \begin{equation*} 2-1=-1 \end{equation*}
      \begin{equation*} -1 \notin \N \end{equation*}
    • Multiplication - take any two \(\N\) and multiply them. Is the result always in \(\N\text{?}\)
      • If yes, \(\N\) are closed under multiplication
      • If no, \(\N\) are not closed under multiplication and a counter-example needs to be provided
      Answer.
      Yes.
    • Division - take any two \(\N\) and divide them. Is the result always in \(\N\text{?}\)
      • If yes, \(\N\) are closed under division
      • If no, \(\N\) are not closed under division and a counter-example needs to be provided
      Answer.
      No.
      \begin{equation*} 1,2 \in \N \end{equation*}
      \begin{equation*} 1/2 = 0.5 \end{equation*}
      \begin{equation*} 0.5 \notin \N \end{equation*}
    • Negation - take any \(\N\) and make it negative. Is the result always in \(\N\text{?}\)
      • If yes, \(\N\) are closed under negation
      • If no, \(\N\) are not closed under negation and a counter-example needs to be provided
      Answer.
      No.
      \begin{equation*} 1 \in \N \end{equation*}
      \begin{equation*} 1 * -1 = -1 \end{equation*}
      \begin{equation*} -1 \notin \N \end{equation*}
    Some additional operations could be: reciprocals, modulus, exponentiation, and factorial.
    If a set is closed under certain operations is helpful to know for type safety as a developer and in proofs.

    Exploration 9. Implications.

    We know \(\N\) are closed under addition. This means we know that if \(x \in \N\) and \(y \in \N\) then \(x+y \in \N\text{.}\)
    This is an example of an implication.
    \begin{equation*} \mbox{ if A then B} A \longrightarrow B \mbox{ or if A is true, then B is true} \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} A \longrightarrow B \mbox{ or if A is true, then B is true} \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} \mbox{ if A is true, then B is true} \end{equation*}

    Exploration 10. Logic in Closure.

    Here is another binary operator:
    \begin{equation*} \mbox{if A and B then C} \end{equation*}
    Given the above logic table, we now know that \(\mbox{if A and B then C} \) is logically equivalent to \(\bar{A} + \bar{B} + C\text{.}\)
    Let us say we know \(C\) is not true and \(B\) is true. What must \(A\) be?
    Answer.
    Not true.
    That is a form of the contrapositive.
    \begin{equation*} A \longrightarrow B \mbox{ means } \bar{A}+B \end{equation*}
    \begin{equation*} \bar{B} \longrightarrow \bar{A} \mbox{ means } \bar{A}+B \end{equation*}
    For this situation, \(AB\longrightarrow C\) means \(\bar{AB} + C = \bar{A}+\bar{B}+C\) and \(\bar{C} \longrightarrow \bar{AB}\) means \(C+\bar{AB} = \bar{A}+\bar{B}+C\text{.}\) So, these statements are the same, but if we know \(\bar{C}\text{,}\) then either A or B is untrue.
    This is telling us that given \(x + y = z\text{,}\) the boolean \(\bar{A}+\bar{B}+C\) must be true. So,
    \begin{equation*} x \notin \N \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} y \notin \N \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} x + y \in \N \end{equation*}
    These conclusions are very important for understanding closure.

    Section 8.2 How to Write a Closure Proof

    These three posibilities are the most important things when writing closure proofs:
    \begin{equation*} x \notin \N \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} y \notin \N \end{equation*}
    \begin{equation*} or \end{equation*}
    \begin{equation*} x + y \in \N \end{equation*}
    Let us say we are given \(x+y\in \R\text{,}\) \(x+y \notin \N\text{,}\) and \(y \in \N\text{.}\)
    • What do we know about \(x\text{?}\)
    What if we know \(x+y \in \N\) and \(x \in \N\text{?}\)
    • What do we know about \(y\text{?}\)
    What if we know \(x+y \in \R\) and \(x \in \N\text{?}\)
    • What do we know about \(y\text{?}\)
    Within all of the scenarios, the arguments must follow the important possibilities.

    Section 8.3 Closure Proven through Construction

    Regular Languages are closed under many operations. Closure can be proven using the construction proof technique.

    RLs Closed under Union.

    RL closure under union follows the basic progression: if \(L_1\) is a RL and \(L_2\) is a RL, then \(L_1\cup L_2\) is a RL.
    If \(L_1\) is a RL, some DFA describes it. It has a start state and a set of accept states. We also added empty string tranistions from the original accept states to a singluar new accept state.
    \begin{equation*} L(M_1) = L_1 \end{equation*}
    This is the same for \(L_2\text{.}\)
    \begin{equation*} L(M_2) = L_2 \end{equation*}
    Using these two machines, we can create the union machine by making a new start state with empty string transitions to the start states for \(M_1\) and \(M_2\text{.}\)
    \begin{equation*} L(M_3) = L_1 \cup L_2 \end{equation*}
    Since this union machine is an NFA, it describes a RL.

    Exploration 11. RLs Closed under Concatenation.

    If \(L_1\) is a RL and \(L_2\) is a RL, then \(L_1 \cdot L_2\) is a RL. This means all strings created by first choosing a string in \(L_1\) and then choosing a string in \(L_2\text{.}\) So,
    \begin{equation*} L_1 \cdot L_2 = \{{xy \| x\in L_1, y\in L_2}\} \end{equation*}
    Hint.
    A good way to approach this is to start where we did last time, by drawing out machines for each language.

    Exploration 12. RLs Closed under Repitition.

    Suppose \(L_1^*\text{,}\) concatenating L_1 with itself 0 or more times.
    Hint.
    Use an empty string transition as a loop.
    Answer.
    Closure by complementation contains an interesting happening. \(\bar{L_1}\) is all strings that can be made with the alphabet but are not in \(L_1\text{.}\) In a DFA, when we’re done processing a string, we are either in an accept state or not. So, \(\bar{L_1}\) can be created by inverting the accept and reject states.

    Exploration 13. RLs Closed under Intersection.

    Intersection \(\cup\) of \(L_1\cup L_2\) is any string that is accepted in \(L_1\) and \(L_2\text{.}\) It is very useful in closure proofs and in crafting NFAs for languages that use the logic of \(and\text{.}\) For example look at these two machines:
    \begin{equation*} L_1(M_1) = \{\mbox{all strings that end with a 1}\} \end{equation*}
    \begin{equation*} L_2(M_2) = \{\mbox{all strings with an even number of symbols}\} \end{equation*}
    \begin{equation*} L_1\cup L_2 = \{\mbox{is any string that ends in a 1 and that has an even number of symbols.}\} \end{equation*}
    Try traceing the path through both machines and only accept if both machines accept. In a more systematic way, consider the states \(Q_1\times Q_2\) and create those machines.
    Hint.
    Here are some examples of the step through thinking.
    • States with \(B_2\) occur when the string has even length.
    • \(A_2\) is when it is odd
    • \(A_1\) says the last seen symbol was a 0
    • \(B_1\) is when it was a 1
    So, it turns out we can use this to build DFAs of challenging languages!
    RLs are closed under other operatons too, like reversal. Their closure can be proven using the methods above or through counterexamples.
